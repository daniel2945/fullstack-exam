# 🚀 השרת החכם לניהול משימות (שאלה 5)

היי! ברוכים הבאים לבאק-אנד של מערכת ניהול המשימות. 
במקום לכתוב הכל בקובץ אחד מבולגן, בניתי פה ארכיטקטורה אמיתית של שרת (MVC) - חילקתי את הקוד למודלים (הנתונים), קונטרולרים (המוח שמחשב הכל) וראוטים (הנתיבים של השרת). הכל מחובר ל-MongoDB Atlas.

השקעתי פה המון מחשבה מעבר לדרישות הבסיסיות של "ליצור ולמחוק משימה". הנה **5 בונוסים** מגניבים שהוספתי למערכת:

## ✨ מה מיוחד בשרת הזה? (הבונוסים)

1. **אבטחה (JWT):** אי אפשר סתם ככה לגשת למשימות. קודם צריך להירשם, להתחבר, ולקבל מהשרת "כרטיס כניסה" דיגיטלי (Token). מי שאין לו כרטיס בתוקף – נחסם מיד.
2. **עבודת צוות והרשאות חכמות:** יצרתם משימה? אתם ה"בעלים" שלה. אבל, אתם יכולים לשתף אותה עם חברים! חבר ששותף במשימה יוכל לראות אותה ולעדכן אותה, אבל **רק אתם תוכלו למחוק אותה**. מנגנון ההרשאות מגן עליכם.
3. **משימות שחוזרות על עצמן:** יש לכם לשלם חשבון פעם בחודש? תגדירו את המשימה כ"חודשית". ברגע שתסמנו אותה כ"הושלמה", השרת אוטומטית ייצר לכם משימה חדשה לגמרי שקופצת בדיוק חודש קדימה.
4. **שומר סף נגד ספאמרים (Rate Limiting):** כדי שאף אחד לא יפיל לנו את השרת עם מיליון בקשות בשנייה (DDoS), הוספתי הגנה שחוסמת אוטומטית כל משתמש שמגזים ושולח מעל 100 בקשות ברבע שעה.
5. **מעקב ורישום (Logging):** כל לחיצה, בקשה או שגיאה שקורית בשרת, מודפסת מיד בצורה מסודרת וצבעונית בחלון הטרמינל כדי שיהיה קל לעקוב אחרי מה שקורה.

---

## 🛠️ איך מריצים את השרת על המחשב?

זה פשוט. פותחים את הטרמינל בתוך התיקייה הזו, ועושים את 3 השלבים הבאים:

1. **התקנת הספריות:** להריץ את הפקודה `npm install` (זה יוריד את כל הספריות הדרושות).
2. **הגדרות סודיות:** ליצור קובץ בשם `.env` בתיקייה הראשית של השרת, ולהגדיר בו את משתני הסביבה הבאים (תחליפו את הערכים בנתונים שלכם):
   ```env
   PORT=<הפורט_להרצת_השרת_למשל_3000>
   MONGO_URI=<מחרוזת_החיבור_שלכם_למונגו_MongoDB_URI>
   JWT_SECRET=<מילת_סוד_אקראית_לאבטחת_הטוקנים>
3. להריץ npm run dev.
